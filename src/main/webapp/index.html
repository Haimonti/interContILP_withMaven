Hermes: Consensus Based Algorithms for Relational Streaming Data

version 1.0

Overview

Hermes implements consensus based algorithms for streaming relational data. It requires a Prolog compiler (YAP) and uses Aleph as the ILP engine. The consensus algorithm is implemented in Java. The main features of the program(s) are the following:
(a) The server has a feature file and background knowledge with which it is able to construct a local model. It is capable of receiving client requests in the form of feature files, make an union with its own feature set and generate a new model. The new model is built using the multiplicative update procedure proposed by Littlestone and Warmuth.
(b) On completing an update procedure including construction of the new model, the weights learnt are sent back to the client who then updates its own local model.
